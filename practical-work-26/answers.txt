Основные проблемы legacy PHP-кода: смешение логики и представления (HTML в PHP), отсутствие четкой архитектуры (MVC), глобальные зависимости через global и $_GLOBALS, уязвимости безопасности (SQL-инъекции, XSS), устаревшие функции (mysql_*), сложность тестирования и поддержки из-за спагетти-кода.

Преимущества Laravel перед чистым PHP: встроенная архитектура MVC, мощная система маршрутизации, ORM Eloquent для работы с БД, миграции, автоматическая инъекция зависимостей, шаблонизатор Blade, встроенная аутентификация и безопасности, богатая экосистема пакетов через Composer.

MVC в Laravel: Model (Eloquent) отвечает за данные и бизнес-логику, View (Blade-шаблоны) за представление, Controller обрабатывает запросы и возвращает ответы; маршруты связывают URL с контроллерами, обеспечивая четкое разделение ответственности.

Composer — менеджер зависимостей для PHP, который управляет установкой и автозагрузкой пакетов через composer.json, обеспечивает контроль версий и позволяет использовать компоненты из экосистемы Packagist для ускорения разработки.

Blade предоставляет наследование шаблонов через @extends, секции (@yield/@section), директивы условных операторов и циклов, подключение подшаблонов, экранирование вывода для безопасности и возможность расширения пользовательскими директивами.

Маршрутизация в Laravel организуется через файлы routes/, где определяются эндпоинты с привязкой к контроллерам или замыканиям, с поддержкой middleware, параметров URL, групп маршрутов и RESTful-ресурсов.

Безопасность в Laravel: защита от SQL-инъекций через Eloquent, встроенная валидация и экранирование вывода в Blade, CSRF-токены для форм, хеширование паролей (bcrypt), шифрование данных, middleware для аутентификации и авторизации политик.

Передача данных в шаблоны происходит через метод view() контроллера: return view('template', ['data' => $value]) или compact(), данные становятся доступны в Blade-шаблонах как переменные, с возможностью проверки существования через isset.
